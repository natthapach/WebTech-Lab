How a web browser builds and displays a web page

 
 

What is client browser in today

ถ้าจะกล่าวถึง web browser ในทุกวันนี้ ควรจะเรียกว่า client browser จะครอบคลุมกว่า เพราะทุกวันนี้ web browser ไม่ได้มีอยู่แค่ใน desktop, client browser นั้นได้แก่ desktop web browser ซึ่งทำงานอยู่บนเครื่อง desktop, mobile web browser ซึ่งแสดงผลบน smart phone, application ที่ฝังตัวอยู่ใน browser ก็เช่นกัน, นอกจากนี้ ทุกวันนี้เครื่องใชต่างๆก็สามารถเป็น client browser ได้ เช่น โทรทัศน์, ตู้เย็น หรือ รถ

How to tell what do you want

การจะระบุสิ่งที่ต้องการจะให้ browser แสดงผลนั้น สามารถทำได้โดยใช้ URL (Uniform Resource Locator) ซึ่งรูปแบบของ url จะเป็นดังนี้

scheme://domain:port/path?query_string#fragment_id

โดยประกอบไปด้วย
  1. scheme

    scheme ที่ใช้ในการส่งข้อมูล เช่น http, https, ftp

  2. domain

    domain ของ server ที่มีข้อมูลที่ต้องการ อาจระบุเป็น domain name หรือ ip address ก็ได้ โดยแต่ล่ะ domain จะมี top-level domain ด้วย เช่น org, edu หรือ com โดยคั่นด้วย .

  3. Port (default 80)

    port number ของ server ที่จะรับคำร้องขอ โดยปกติแล้วจะมีค่าเท่ากับ 80 แต่ถ้าหากต้องการเข้าถึง application บางอย่าง เช่น Rail ต้องใช้ port 3000 นอกจากนี้การระบุ port ยังเหมาะสำหรับการ run หลาย web service บนเครื่องเดียวกัน

  4. Path

    สำหรับระบุ path ในการเข้าสู่ไฟล์ที่ต้องการ อาจเกิดจากการรวมกันของ folder หลายๆชั้นก็ได้ สุดท้ายแล้วต้องตามด้วยนามสกุลของไฟล์ เช่น html ด้วยการระบุแบบนี้ จะทำให้ server รู้ว่าจะต้องไปนำไฟล์มาจากที่ไหนเพื่อส่งกลับมาให้

  5. Query String

    เป็น optional สามารถไม่ระบุได้

นอกจาก URL ที่ระบุให้ browser ไปแล้วนั้น ยังมี URL ที่อยู่ภายในไฟล์ HTML ที่ได้รับมาจาก server ซึ่งได้มาจาก URL ที่ให้ไปนั่นเอง

URL ภายใน HTML นั้นจะแบ่งเป็น 2 แบบ

  1. Use for this page

    เป็น URL ที่ระบุไปยัง resource ที่ต้องใช้เพื่อแสดงผลหน้าเว็บดังกล่าวผ เช่น ไฟล์ CSS, javascript, image โดย URL ประเภทนี้ browser จะไปโหลดมาให้โดยอัตโนมัติ

  2. Hyperlinks

    เป็น URL สำหรับระบุตำแหน่งของ page อื่น โดยจะโหลดก็ต่อเมื่อผู้ใช้คลิกเท่านั้น

The step to load HTML

สรุปขั้นตอนการจะได้มาซึ่ง HTML resouce สำหรับแสดงผลหน้าเว็บ

  1. ผู้ใช้ระบุ URL ลงไปใน address bar

  2. client browser จะใช้ Hypertext Protocol ผ่านคำสั่ง GET (หมายถึงว่าต้องการ resource ตาม URL ที่ระบุมานี้)

  3. command นี้จะถูกส่งออกไปตาม network เพื่อหาเครื่อง server ที่ระบุไว้ใน domain

  4. server เมื่อได้รับ request จะไปตามหา resource ที่ระบุไวใน Path โดยอาจใช้ database system หรือ file system

  5. ถ้าหาก server หาเจอ จะส่ง resource นั้นกลับไปยัง client

  6. เมื่อ client ได้รับ HTML แล้ว ก็จะดำเนินการประมวลผล

Processing HTML

เมื่อได้รับ HTML มา browser จะทำการ parse(การแบ่งไฟล์ออกเป็นชิ้นๆ และดำเนินการกับแต่ละชิ้นนั้นๆ) HTML จะประกอบไปด้วย content และ structure ของหน้าเว็บ

นอกจากนี้ HTML ก็จะมี URL เพื่อระบุตำแหน่งของ resource อื่นเพื่อใช้ในการแสดงผล เช่น ถ้าหากมี URL สำหรับรูปภาพ browser ก็ต้องไปตาม URL นี้เพื่อนำมาแสดงผล นอกจากนี้ก็ยังมี resource อื่นๆที่ใช้ในการแสดงผล เช่น CSS, javascript

Cascading Style Sheet(CSS) เป็นไฟล์สำหรับกำหนดรูปแบบ, หน้าตา ของเนื้อหาที่จะแสดงบนหน้าเว็บ ส่วน javascript นั้นจะกำหนดพฤติกรรม,การทำงานของหน้าเว็บ เช่นหากมีการกดปุ่ม submit ก็จะรวบรวมข้อมูลทั้งหมดใน form ส่งไป

How javascript interact elements

browser จะทำการรวบรวม resource ต่างๆเข้าด้วยกันเป็น document และเก็บไว้ใน internal data structures เมื่อต้องการแสดงผล browser จะนำ document นี้มา render(การแสดงผลตามที่ระบุใน document ลงในหน้าจอ)

นอกจากนี้ browser ยังได้ทำการ publish document ให้ออกมาในรูปของ Document object model ซึ่ง javascript จะใช้ model นี้เพื่อควบคุมการทำงานต่างๆของหน้าเว็บ

browser ไม่ได้รัน javascript แต่ละอันตามลำดับ แต่จะรันทันทีที่หาเจอ ดังนั้น javascript programmer ที่ดีคือการเขียน script ที่สามารถรันได้เมื่อทุกอย่างพร้อม
